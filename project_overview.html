The Strategy
CE Leg — SELL Call
OTM Call above ATM
e.g. NIFTY 23850 CE @ ₹192
Profit if NIFTY stays below 23850
PE Leg — SELL Put
OTM Put below ATM
e.g. NIFTY 23750 PE @ ₹186
Profit if NIFTY stays above 23750
PROFIT ZONE — NIFTY stays between the two strikes
23750 PE
23850 CE
Combined Premium = ₹192 + ₹186 = ₹378 collected

How Each Day Works
1
9:30 AM — Signal Engine Scans Every 5 min
Checks all 8 conditions (VIX, margin, holiday, spot stability…). If all pass, finds the best CE + PE strikes with enough premium.
ATM = round(spot/50)×50 → walk up/down until LTP ≥ ₹40
2
Order Placement AngelOne API
SELL CE + SELL PE simultaneously. If one leg fails, the other is immediately bought back (partial fill recovery). On success, premium is locked in.
SELL NIFTY13APR2623850CE qty=195 + SELL NIFTY13APR2623750PE qty=195
3
Position Monitoring Every 30 sec
Fetches live LTP for both legs. Calculates open P&L. At 20% profit, trailing SL locks to breakeven. Watches for target / SL / time exits.
open_pnl = (entry_premium − current_cost) × 195
4
Exit — BUY Back Both Legs
Square off triggered by target hit, SL hit, dead zone (2:30 PM), or EOD (3:20 PM). Telegram alert sent. Trade recorded to local JSON + Google Sheets.
BUY CE + BUY PE → final_pnl saved → active_pos.json deleted
5
Repeat Up to 3 trades/day
After exit, signal engine resumes scanning. Max 3 trades per day or until daily loss limit hit. Agent stops automatically at end of day.

① active_pos.json Priority
Written on every trade entry and when trailing SL activates. Has the full position detail — tokens, entry prices, SL state, trail_locked flag.
Deleted when position closes.
② trades.json Fallback
Contains all trade records. An entry with exit_time = null means trade is still open. Used if active_pos.json is missing.
Always kept in sync — entry + exit both saved here.
On restart: Bot reads active_pos.json → restores position → re-resolves tokens from AngelOne after login → continues monitoring with no data loss.

Telegram Bot Commands
Send commands directly to the bot from your Telegram chat. Only messages from the configured TELEGRAM_CHAT_ID are accepted — all others are silently ignored. No PIN required (chat identity is the auth).
CommandAction
/statusLive NIFTY spot, VIX, bot state, active position, open P&L
/pnlToday's realized P&L and trade count
/runStart the trading agent (begin scanning for signals)
/stopStop the trading agent (no new trades, existing position monitored)
/exitEmergency square-off — closes both CE & PE legs immediately
/approveExecute the pending trade signal (MANUAL mode only)
/mode autoSwitch to AUTO mode — trades execute automatically on signal
/mode manualSwitch to MANUAL mode — wait for /approve before executing
/menuShow the full command list in chat (also shown on /start)
⚙️ How it works: Server long-polls getUpdates every 20 s. No webhook or public URL needed. Polling starts automatically with the server.

File Structure
🐍
dev_server.py
Main server — Flask API + all trading logic (~2,400 lines). Signal engine, order placement, position monitor, P&L, Sheets, Telegram.
🌐
nifty_dashboard.html
Single-file web dashboard. Real-time UI with CE/PE position card, trade log modal, settings panel. PWA-ready.
💾
trades.json
All trade records (open + closed). Survives restarts. Source of truth for trade history and daily P&L recovery.
🔴
active_pos.json
Full snapshot of current open position. Written on entry, updated on trailing SL. Deleted on exit. Used for crash recovery.
🔑
.env
API keys: ANGEL_API_KEY, TELEGRAM_TOKEN, GSHEET_ID, OPENAI_API_KEY. Never committed to git.
📋
gsheet_creds.json
Google Service Account credentials for Sheets API access. Never committed to git.
📱
manifest.json + sw.js
PWA files — makes dashboard installable on mobile (Android + iOS). Service worker caches shell, API always live.
🚀
setup_auto_start.bat
One-time Windows Task Scheduler setup. Runs FIFTO silently on every login. No console window.
