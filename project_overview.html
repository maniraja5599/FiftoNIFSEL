Strikes Selected — Delta-Based (Probability-First)

5
Straddle Premium Filter — IV Environment Check
Compares today's ATM straddle premium against the last 10 recordings at the same DTE (e.g. DTE=3 vs DTE=3 history). If today's premium is less than 80% of that average, IV is too low — skip and wait.
✗ Premium ₹158 < 80% of avg ₹207 → skip   ✓ Premium ₹220 ≥ 80% of avg ₹207 → enter
Needs 5+ trading days of history to activate. Allows all entries while history is thin.

Safety & Reliability Systems
Built-in guards prevent the bot from trading on wrong days, firing duplicate orders, or running while the market is closed. These work silently in the background every session.
🗓️
NSE Holiday Detection
All NSE F&O trading holidays for 2026 are hardcoded as a fallback. On startup the bot also tries to fetch the live NSE holiday list and merges both. If the API fails, the hardcoded set is used — the bot never trades on a holiday regardless of connectivity.
_NSE_HOLIDAYS_2026 dict · 2026-04-14 → "Dr. B.R. Ambedkar Jayanti"
🔒
Duplicate Trade Prevention
Two independent layers stop the same signal firing twice. First, signal_pending = True is set in AUTO mode the moment a signal is generated — any further scans skip entry until the trade completes. Second, a threading lock (_trade_exec_lock) blocks concurrent calls to _execute_trade().
threading.Lock() · non-blocking acquire · signal_pending flag
⏸️
Market-Closed Thread Gates
All three active background threads (signal engine, market-data fetch, fast LTP refresh) check _is_market_open() before doing any work. If the market is closed they sleep and poll — no broker API calls, no LTP reads, no signals generated.
signal_engine: waits at 9:15 · market_data: sleep 300s · ltp: sleep 60s
🟢
Google Sheet Deduplication
Before appending a trade to the Google Sheet log, the bot searches column A for the trade ID. If the row already exists it skips the write and logs a warning. Prevents duplicate rows from appearing after retries or restarts.
ws.find(trade_id, in_column=1) → skip if exists
🔴
Market Closed Reason — Live UI
The dashboard shows why the market is closed: Weekend (Saturday/Sunday), NSE holiday by name, before 9:15 AM, or after 3:30 PM. The server computes this and sends it via /api/state so the UI is always accurate even without the browser knowing about holidays.
market_closed_reason: "NSE Holiday — Dr. B.R. Ambedkar Jayanti"
⚠️
No-Trade Reason — Live UI
During the entry window, if conditions are failing the dashboard shows exactly which checks are blocking entry: VIX out of range, IV too low, EMA trending, PCR out of range, margin insufficient, or any loss gate hit. Updated every poll cycle so you can see live why the bot is waiting.
no_trade_reason: "Checks failing: VIX out of range, IV too low"
