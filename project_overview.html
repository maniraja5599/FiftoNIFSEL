≥10

<10

PE Leg — SELL Put
OTM Put below ATM

23850 CE

Max profit: ₹378 × 195 = ₹73,710  |  Both options expire worthless
② Bull Put Spread Bullish
Sell Leg — SELL OTM Put
Higher strike put (closer to ATM)
e.g. NIFTY 23700 PE @ ₹120
Premium received
Hedge Leg — BUY Lower Put
Lower strike put (deeper OTM)
e.g. NIFTY 23600 PE @ ₹65
Premium paid (caps max loss)
Net Premium = ₹120 − ₹65 = ₹55 collected  |  Max loss capped at spread width − net premium
Target, SL and trail SL are all calculated on the net spread premium (₹55), not the sold leg alone.
③ Bear Call Spread Bearish
Sell Leg — SELL OTM Call
Lower strike call (closer to ATM)
e.g. NIFTY 23900 CE @ ₹100
Premium received
Hedge Leg — BUY Higher Call
Higher strike call (deeper OTM)
e.g. NIFTY 24000 CE @ ₹50
Premium paid (caps max loss)
Net Premium = ₹100 − ₹50 = ₹50 collected  |  Max loss capped at spread width − net premium
Target, SL and trail SL are all calculated on the net spread premium (₹50), not the sold leg alone.
How Each Day Works
1
9:30 AM — Signal Engine Scans Every 5 min
Checks all 16 conditions across 5 groups: pre-conditions → loss gates → market checks → trend detection → strike filters. Determines setup type (Strangle/Bull Put Spread/Bear Call Spread) from EMA trend + PCR. Margin check is display-only and does not block entry.
ATM = round(spot/50)×50 → walk up/down until LTP ≥ ₹40 → OI ≥ 3L → combined ≥ ₹150 (strangle)
2
Order Placement AngelOne API
Three setup types supported. Strangle: SELL CE + SELL PE. Bull Put Spread: SELL PE + BUY lower PE. Bear Call Spread: SELL CE + BUY higher CE. If any leg fails, filled legs are immediately bought back (partial fill recovery). For spreads, the hedge LTP is fetched right after placement to compute the actual net spread premium.
Strangle: SELL 23850CE + SELL 23750PE  |  Bull Put: SELL 23700PE + BUY 23600PE
3
Position Monitoring Every 5 sec (LTP) / 30 sec (SL check)
Fetches live LTP for all legs including hedge. For spreads, current cost = short leg LTP − hedge leg LTP (net). Calculates open P&L. At 20% profit, trailing SL locks to breakeven. Watches for target / SL / time exits.
Strangle: current_cost = CE_ltp + PE_ltp  |  Spread: current_cost = sell_ltp − hedge_ltp
4
Exit — Square Off All Legs
Square off triggered by target hit, SL hit, dead zone (2:30 PM), or EOD (3:20 PM). For spreads: BUY back the sold leg + SELL the hedge leg. Telegram alert sent. Trade recorded to local JSON + Google Sheets.
Spread exit: BUY sold_leg + SELL hedge_leg → net P&L saved → active_pos.json deleted
5
Repeat Up to 3 trades/day
After exit, signal engine resumes scanning. Max 3 trades per day or until daily loss limit hit. Agent stops automatically at end of day.

Entry Conditions — Complete Checklist
Hard block — trade cannot enter Optional — skipped when data unavailable Display only — shown on dashboard, does not block Strike-level filter — runs inside _select_strikes()
① Pre-conditions — Checked Before Any Scan
ConditionRuleTypeDetail
🤖 Agent Running bot_running = True Hard Start/stop button on dashboard. Telegram /run / /stop. No scan happens when stopped.
🏦 Market Open Mon–Fri, 9:15–15:30 Hard Weekday check + time range. Signal engine sleeps 5 min between checks when market is closed. No orders possible outside market hours.
⏰ Entry Window 9:30 AM – 11:00 AM Hard Only scan for new trades in this window. Avoids 9:15–9:30 opening volatility and avoids entering too late (low theta decay benefit). Configurable via entry_start / entry_end.
🔢 Max Trades/Day trades_today < 3 Hard Hard cap at 3 trades per day (configurable: max_trades_per_day). Prevents over-trading after a loss. Counter resets at midnight. Incremented at entry, NOT at exit.
⏱️ Signal Cooldown ≥ 5 min since last signal Hard Signal engine enforces a 5-minute cooldown between signal evaluations. Even if all checks pass, the engine won't fire again until 5 min has elapsed.
📡 Active Position active_position = False Hard No new trade is opened while one is already running. Bot is single-position — one open trade at a time. Signal engine skips entirely if active_position is True.
② Loss Gates — Circuit Breakers
ConditionRuleTypeDetail
📉 Daily Loss Gate daily_pnl > −₹45,000 Hard Daily realized P&L must stay above the loss limit. Configurable: daily_loss_limit. Checked every cycle. Once hit, no new trades for the rest of the day. Gate resets at midnight.
📆 Weekly Loss Gate week_pnl > −₹60,000 Hard Sum of closed P&L from Mon–Sun week. Configurable: weekly_loss_limit. Protects against a run of consecutive losing days. Trades blocked for the rest of the week if hit.
📅 Monthly Loss Gate month_pnl > −₹1,50,000 Hard Sum of closed P&L for the calendar month. Configurable: monthly_loss_limit. Largest safety net — protects capital across the full month.
🔴 Stop-After-SL _sl_hit_today = False Hard When stop_after_sl = True in config, the daily gate is force-set to False after any SL hit. No further trades for that day after a stop loss. Resets at midnight.
📅 NSE Holiday today not in holiday_set Hard Holiday list is fetched from NSE API on startup each day. Falls back to a hard-coded set of known 2026 NSE F&O holidays if the API call fails.
③ Market Condition Checks — Run on Every Scan Cycle
ConditionRuleTypeDetail
🌡️ VIX Range 13 ≤ VIX ≤ 28 Hard India VIX must be in range. Below 13 = premiums too cheap to sell profitably. Above 28 = dangerous gamma, SL hit probability too high. Configurable: vix_min / vix_max. Refreshed every 15 s from AngelOne/Upstox.
📌 Spot Stability |Δspot| < 0.5% in 15 min Hard NIFTY spot is tracked in a rolling history every 15 s. If the absolute % change in the last 15 minutes exceeds spot_momentum_limit (default 0.5%), entry is blocked. Avoids selling into a fast directional move.
⚖️ PCR Range 0.8 ≤ PCR ≤ 1.4 Optional Put-Call Ratio from NSE option chain. Range [0.8, 1.4] means sentiment is not extreme. Passed when data is unavailable (None = skip). If PCR is outside range it means strong directional bias — skip neutral setups. Also used for setup-type selection: PCR ≥ 1.0 = bullish bias → Bull Put Spread; PCR ≤ 1.0 = bearish → Bear Call Spread.
📊 IV Percentile IVP > 40 (or IV_ATM > 10%) Optional Primary: 30-day rolling historical percentile from iv_history.json. Pass = IVP > 40 (current IV is in top 60% of recent history). When < 10 days of history exist, falls back to raw ATM IV > 10%. When no IV data at all: skipped (None). Ensures we only sell options when IV is relatively elevated and premiums are worth collecting.
📈 EMA Trend / Flat Depends on trend Optional EMA 9 vs EMA 21 on 15-min candles + ATR comparison.
FLAT: spread ≤ 0.20× ATR & slope_gap ≤ 0.08× ATR → Short Strangle allowed.
UP: EMA9 > EMA21, both sloping up, spread > 0.30× ATR → Bull Put Spread.
DOWN: EMA9 < EMA21, both sloping down, spread > 0.30× ATR → Bear Call Spread.
For FLAT trend, if EMA check returns False (not flat), strangle entry is blocked. For UP/DOWN, the EMA check is skipped in _all_checks_pass (trend already known) but PCR must confirm direction.
💰 Margin Available avail_cash > 0 Display Only Available margin is fetched from AngelOne. Shown on the dashboard as an informational check but does not block entry. Broker will reject the order at placement time if margin is insufficient — that failure is handled by the partial fill recovery logic.
④ Trend Detection → Setup Type Selection
Before calling strike selection, the signal engine determines which setup to trade based on EMA trend + PCR confirmation. This happens every scan cycle once all checks above pass.
FLAT Trend
Condition: EMA spread ≤ 0.20× ATR
→ Short Strangle
EMA flat check enforced.
If EMA is not flat → skip this cycle.
UP Trend + PCR ≥ 1.0
EMA9 > EMA21, both slopes positive
→ Bull Put Spread
SELL OTM PE + BUY PE 100pts lower.
PCR ≥ 1 confirms bullish sentiment.
DOWN Trend + PCR ≤ 1.0
EMA9 < EMA21, both slopes negative
→ Bear Call Spread
SELL OTM CE + BUY CE 100pts higher.
PCR ≤ 1 confirms bearish sentiment.
Fallback: If trend is UP/DOWN but PCR doesn't confirm (e.g. trend UP but PCR < 1.0), falls back to Short Strangle — but only if EMA is flat. If EMA is not flat in this case, the cycle is skipped entirely.
⑤ Strike Selection Filters — Run Inside _select_strikes()
These filters apply after the setup type is chosen. They validate whether valid strikes exist with enough premium and liquidity. If any fails, no signal is generated for that cycle.
FilterRuleApplies ToDetail
💵 Min Leg Premium LTP ≥ ₹40 All setups Each leg's LTP must be ≥ min_premium (₹40). Walk OTM in ₹50 strike steps from ATM until a qualifying strike is found (up to ₹500 away). If no strike qualifies, no signal.
💎 Min Combined Premium CE_LTP + PE_LTP ≥ ₹150 Strangle only Combined CE + PE premium must be ≥ min_combined_premium (₹150). Ensures the strangle collects enough premium to cover brokerage and still profit. Not applied to spreads (only one leg is sold).
📊 OI Filter OI ≥ 3,00,000 Per-setup leg only Open Interest at the selected strike must be ≥ min_strike_oi (3 lakh contracts). Ensures adequate liquidity — low OI = slippage risk. Strangle: both CE and PE must pass. Bull Put Spread: only PE checked. Bear Call Spread: only CE checked.
📈 20-Day Straddle Avg ATM_straddle ≥ 80% of 20d avg Strangle only ATM straddle (CE + PE at ATM) is compared against the 20-day rolling average premium stored in straddle_history.json. If today's straddle is < 80% of the average, premiums are abnormally cheap — entry blocked. Today's premium is also recorded for future averaging. Skipped for spread setups.
ATM calculation: ATM = round(spot / 50) × 50. Strikes walk from ATM+50 upward for CE, ATM−50 downward for PE, in ₹50 increments. First strike with LTP ≥ ₹40 is selected.
Target & Stop Loss — DTE Tiered
Risk/reward auto-adjusts based on days remaining to expiry. More DTE = conservative targets.
Spreads: All percentages apply to the net spread premium (sell leg LTP − hedge leg LTP). Example: SELL PE ₹120 − BUY PE ₹65 = net ₹55. Target & SL calculated on ₹55, not ₹120.

🔒 Trailing SL — Breakeven Lock
When profit reaches 20% of premium collected, stop loss automatically tightens to breakeven (= entry premium). Worst case after trail: ₹0 loss. For spreads, "entry premium" is the net spread premium (sold leg − hedge leg).
Strangle: Net ₹378 × 20% = ₹75.6 → SL locks at ₹378  |  Spread: Net ₹55 × 20% = ₹11 → SL locks at ₹55

Telegram Bot Commands
Send commands directly to the bot from your Telegram chat. Only messages from the configured TELEGRAM_CHAT_ID are accepted — all others are silently ignored. No PIN required (chat identity is the auth).
CommandAction
/statusLive NIFTY spot, VIX, bot state, active position, open P&L
/pnlToday's realized P&L and trade count
/runStart the trading agent (begin scanning for signals)
/stopStop the trading agent (no new trades, existing position monitored)
/exitEmergency square-off — closes both CE & PE legs immediately
/approveExecute the pending trade signal (MANUAL mode only)
/mode autoSwitch to AUTO mode — trades execute automatically on signal
/mode manualSwitch to MANUAL mode — wait for /approve before executing
/menuShow the full command list in chat (also shown on /start)
⚙️ How it works: Server long-polls getUpdates every 20 s. No webhook or public URL needed. Polling starts automatically with the server.

File Structure
🐍
dev_server.py
Main server — Flask API + all trading logic (~3,700 lines). Signal engine, order placement, position monitor, fast LTP refresh, P&L, Sheets, Telegram.
🌐
nifty_dashboard.html
Single-file web dashboard. Real-time UI with CE/PE position card, trade log modal, settings panel. PWA-ready.
💾
trades.json
All trade records (open + closed). Survives restarts. Source of truth for trade history and daily P&L recovery.
🔴
active_pos.json
Full snapshot of current open position. Written on entry, updated on trailing SL. Deleted on exit. Used for crash recovery.
📈
iv_history.json
Rolling 30-day ATM IV history (one entry per trading day). Used to compute IV percentile for the entry check. Falls back to raw ATM IV check when < 10 days exist. Keeps last 60 days.
🔑
.env
API keys: ANGEL_API_KEY, TELEGRAM_TOKEN, GSHEET_ID, OPENAI_API_KEY. Never committed to git.
📋
gsheet_creds.json
Google Service Account credentials for Sheets API access. Never committed to git.
📱
manifest.json + sw.js
PWA files — makes dashboard installable on mobile (Android + iOS). Service worker caches shell, API always live.
🚀
setup_auto_start.bat
One-time Windows Task Scheduler setup. Runs FIFTO silently on every login. No console window.
